Local Architecture
•	Services: app (Django), db (PostgreSQL)
•	Port Mapping: Host 8000 -> App 8000
•	Volumes: dev-db-data (database), dev-static-data (static/media)
•	Networking: Browser -> app -> db (DB_HOST=db)
•	Development startup: wait_for_db -> migrate -> runserver
Production Architecture
•	Services: proxy, app, db
•	Public Entry: Host port 80 -> Proxy
•	Proxy forwards requests to app:9000 using uWSGI.
•	App startup: wait_for_db -> collectstatic -> migrate -> uWSGI
•	Shared static-data volume between app and proxy.
•	Database persistence via postgres-data volume.
•	Environment values injected using .env variables.
